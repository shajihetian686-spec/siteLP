# 設計判断記録

## 2026-04-20: プロジェクト初期構築 - カラースキーム・レイアウト方針

### 判断者
- エージェント: Builder

### 背景
- Web制作サービスのLPデザイン完成イメージ（画像）を忠実にHTML/CSSで再現する
- デザイン提案ではなく再現が目的

### 選択肢
1. 案A: フレームワーク（Bootstrap/Tailwind）を使用
2. 案B: バニラCSS（フレームワークなし）で実装

### 決定
- 採用案: 案B（バニラCSS）
- 理由: デザインの細かな再現にはフレームワークの制約が邪魔になる。余白・角丸・シャドウなど、画像に合わせた微調整が必要なため、CSS変数を使ったバニラCSSが最適。

## 2026-04-20: カラースキーム決定

### 判断者
- エージェント: Builder

### 背景
- 完成イメージ画像から配色を抽出する必要がある

### 決定
- メインカラー（濃紺）: #1B2A4A
- アクセント（オレンジ）: #E8792E
- ベース: #FFFFFF
- 補助背景: #F7F7F7（薄グレー）, #FAF8F5（薄ベージュ）
- テキスト: #333333, #666666
- 理由: 画像のトーンに合わせ、落ち着いた濃紺とビビッドすぎないオレンジを選定。

## 2026-04-20: アニメーション方針

### 判断者
- エージェント: Builder

### 背景
- JavaScriptは最小限という要件

### 選択肢
1. 案A: AOS.jsをCDNで導入
2. 案B: Intersection Observerで自前実装
3. 案C: アニメーションなし

### 決定
- 採用案: 案B（Intersection Observer）
- 理由: 外部依存なしで軽量。カード要素のフェードイン程度の控えめなアニメーションで、上品な印象を保つ。

## 2026-04-20: フォント選定

### 判断者
- エージェント: Builder

### 決定
